# Docker: очистка устаревшего без потери данных запущенных контейнеров

Можно безопасно почистить **всё, что не используется запущенными контейнерами**, не трогая их данные.

## 1) Зафиксируй, что «нужно оставить»

Покажи, что сейчас запущено:

```bash
docker ps
```

Для каждого нужного контейнера посмотри, **какие volumes реально подключены**:

```bash
docker inspect -f '{{.Name}} -> {{range .Mounts}}{{.Type}}:{{.Name}}{{" ("}}{{.Destination}}{{") "}}{{end}}' $(docker ps -q)
```

(Выведет имена named-volumes; их трогать нельзя.)

## 2) Удалить всё остановленное/лишнее (без удаления volume'ов)

Это удалит **остановленные контейнеры, неиспользуемые сети, dangling-образы и build-кэш**. **Volumes не затронет**.

```bash
docker system prune -f
```

Если хочешь также снести **все неиспользуемые образы** (кроме тех, что нужны запущенным контейнерам):

```bash
docker system prune -a -f
```

## 3) Volumes (осторожно)

Удалять volumes стоит только если уверен. Команда ниже удалит **только «осиротевшие» volumes**, которые **не подключены ни к одному контейнеру** (в т.ч. остановленному).

Сначала посмотри, что будет удалено:

```bash
docker volume ls
docker volume prune --filter label!=keep --dry-run 2>/dev/null || true
```

На практике обычно делают так: **помечают нужные volumes меткой** и чистят всё остальное.  
Если у твоих volume'ов нет меток, лучше так:

- сначала просто список «висячих» volume'ов:

```bash
docker volume ls -qf dangling=true
```

- затем (если список точно безопасен):

```bash
docker volume prune -f
```

## 4) Build cache отдельно (часто съедает много)

Это безопасно для данных контейнеров:

```bash
docker builder prune -f
```

или более агрессивно (всё, что не нужно):

```bash
docker builder prune -a -f
```

## 5) Проверка, что данные «живого» не затронуты

- `docker system prune` **не удаляет запущенные контейнеры**
- `docker volume prune` **не удаляет volumes, которые подключены к хотя бы одному контейнеру**

---

Если нужна максимально безопасная проверка перед агрессивной очисткой: пришли вывод `docker ps` и результат команды из пункта 1 (`inspect` mounts) — по ним видно, какие volumes и образы «живые».
